<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> affiliate APIs | Untrusted provider and activity parameters |
| API -> external booking providers | Redirect URL integrity and tracking continuity |
| Tracking storage -> analytics/reporting | Integrity of attribution data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | Affiliate link generation | mitigate | Provider allowlist and strict URL construction in server code; no raw URL passthrough. |
| T-04-02-02 | Information disclosure | Click tracking records | mitigate | Store only required attribution fields and avoid sensitive payloads in redirect params. |
| T-04-02-03 | Repudiation | Affiliate click attribution | mitigate | Persist immutable click token + timestamp before redirect issuance. |
| T-04-02-04 | Open redirect abuse | Click endpoint | mitigate | Verify redirect target belongs to known provider domains before response. |
</threat_model>
