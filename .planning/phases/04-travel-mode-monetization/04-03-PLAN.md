<threat_model>

## Trust Boundaries

| Boundary                      | Description                                 |
| ----------------------------- | ------------------------------------------- |
| Client -> billing APIs        | Untrusted checkout and event requests       |
| Stripe -> webhook endpoint    | External event authenticity and replay risk |
| Client -> analytics ingestion | Potential event spam and PII leakage        |

## STRIDE Threat Register

| Threat ID  | Category               | Component          | Disposition | Mitigation Plan                                                                |
| ---------- | ---------------------- | ------------------ | ----------- | ------------------------------------------------------------------------------ |
| T-04-03-01 | Spoofing               | Billing webhook    | mitigate    | Verify Stripe signature and reject unsigned/invalid events.                    |
| T-04-03-02 | Replay attack          | Webhook processing | mitigate    | Enforce idempotency by event id tracking before mutation.                      |
| T-04-03-03 | Tampering              | Entitlement state  | mitigate    | Derive entitlements server-side only from subscription records.                |
| T-04-03-04 | Information disclosure | Analytics endpoint | mitigate    | Validate allowlisted properties and strip sensitive fields before persistence. |

</threat_model>
