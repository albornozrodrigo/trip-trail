<threat_model>

## Trust Boundaries

| Boundary                                 | Description                                  |
| ---------------------------------------- | -------------------------------------------- |
| Browser service worker -> cached content | Potential stale or manipulated offline state |
| Client -> alert scheduling API           | Untrusted schedule payloads                  |
| API -> trip data                         | Authorization for offline snapshot access    |

## STRIDE Threat Register

| Threat ID  | Category               | Component                        | Disposition | Mitigation Plan                                                                  |
| ---------- | ---------------------- | -------------------------------- | ----------- | -------------------------------------------------------------------------------- |
| T-04-01-01 | Information disclosure | Offline snapshot API             | mitigate    | Enforce trip membership checks and return minimal offline fields only.           |
| T-04-01-02 | Tampering              | Service worker cache             | mitigate    | Use versioned cache keys and checksum metadata for snapshot payloads.            |
| T-04-01-03 | Denial of service      | Alert scheduling endpoint        | mitigate    | Validate payload size and apply per-user rate limits on scheduling calls.        |
| T-04-01-04 | Spoofing               | Notification scheduling messages | mitigate    | Accept only structured messages from trusted app origin with strict type guards. |

</threat_model>
