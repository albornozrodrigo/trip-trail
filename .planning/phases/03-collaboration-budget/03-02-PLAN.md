<threat_model>

## Trust Boundaries

| Boundary                           | Description                                   |
| ---------------------------------- | --------------------------------------------- |
| Collaborator input -> comments API | Untrusted text and thread references          |
| Collaborator action -> vote API    | Potential vote manipulation and spam          |
| Notification events -> client UI   | Event payload integrity and recipient privacy |

## STRIDE Threat Register

| Threat ID  | Category               | Component              | Disposition | Mitigation Plan                                                                                     |
| ---------- | ---------------------- | ---------------------- | ----------- | --------------------------------------------------------------------------------------------------- |
| T-03-02-01 | Tampering              | Vote endpoint          | mitigate    | Enforce one-row upsert per collaborator/activity and validate collaborator membership before write. |
| T-03-02-02 | Information disclosure | Notifications endpoint | mitigate    | Filter by recipient collaborator id and trip membership; no cross-trip payload leak.                |
| T-03-02-03 | Repudiation            | Comment edits/deletes  | mitigate    | Persist author id + timestamps and soft-delete marker for auditability.                             |
| T-03-02-04 | Denial of service      | Comment submit path    | mitigate    | Rate-limit comment creation per collaborator and cap content length.                                |

</threat_model>
