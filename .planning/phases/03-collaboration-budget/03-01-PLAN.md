<threat_model>

## Trust Boundaries

| Boundary                                | Description                                              |
| --------------------------------------- | -------------------------------------------------------- |
| Invite link -> API                      | Untrusted token and guest identity enter server boundary |
| Collaborator client -> realtime channel | Presence events from potentially malicious clients       |
| API -> database                         | Access control and token lifecycle enforcement           |

## STRIDE Threat Register

| Threat ID  | Category               | Component                | Disposition | Mitigation Plan                                                                            |
| ---------- | ---------------------- | ------------------------ | ----------- | ------------------------------------------------------------------------------------------ |
| T-03-01-01 | Spoofing               | Invite join endpoint     | mitigate    | Hash tokens at rest, enforce expiry/max-use, bind guest cookie to trip and rotate on join. |
| T-03-01-02 | Tampering              | Collaborator role writes | mitigate    | RLS denies role escalation; owner-only role mutation path.                                 |
| T-03-01-03 | Denial of service      | Presence heartbeat       | mitigate    | Throttle heartbeat publishes and ignore over-frequency events server-side.                 |
| T-03-01-04 | Information disclosure | Collaborator roster API  | mitigate    | Only trip members can read roster; return minimal profile fields.                          |

</threat_model>
